Available algorithms

Bitonic Sort

A network-based sort built for parallel environments rather than everyday general-purpose use.

O(n log² n) Parallel Friendly Memory: O(n) Advanced

Bubble Sort

A simple adjacent-swap algorithm that is easy to understand but inefficient on larger inputs.

Best: O(n) Avg: O(n²) Memory: O(1) Stable Classic

Bucket Sort

Groups values into buckets first, then sorts within each bucket for near-linear performance on suitable data.

O(n + k) Distribution Sort Memory: O(n) Non Comparative

Counting Sort

Counts occurrences instead of comparing elements, making it fast when the value range is constrained.

O(n + k) Non Comparative Stable Memory: O(k)

Heap Sort

Turns the array into a heap to guarantee O(n log n) time while staying in place.

O(n log n) Memory: O(1) In Place Binary Heap

Insertion Sort

Builds the sorted portion one item at a time and works especially well on small or nearly sorted data.

Best: O(n) Avg: O(n²) Memory: O(1) Stable

Intro Sort

Starts as quick sort and switches strategies when needed to avoid worst-case performance.

O(n log n) Hybrid Memory: O(log n) Industry Standard

Merge Sort

Recursively splits the array and merges sorted halves for reliable O(n log n) performance.

O(n log n) Memory: O(n) Stable Divide & Conquer

Merge Sort (In Place)

A memory-optimized merge variant that trades implementation simplicity for in-place behavior.

O(n log n) Memory: O(1) Advanced

Quick Sort

Partitions around a pivot and is usually one of the fastest comparison sorts in practice.

O(n log n) Worst: O(n²) Memory: O(log n) Fast in Practice

Radix Sort

Sorts by individual digits or characters, avoiding direct comparisons entirely.

O(nk) Non Comparative Memory: O(n + k)

Selection Sort

Repeatedly selects the next smallest value, minimizing writes but not comparisons.

O(n²) Memory: O(1) Classic

Shell Sort

Improves insertion sort by comparing elements across shrinking gaps before the final pass.

O(n log² n) In Place Gap Based

Tim Sort

A production-grade hybrid sort that exploits natural runs and uses merge-like behavior underneath.

O(n log n) Hybrid Stable Python Default

Tournament Sort

Uses a tournament tree to repeatedly surface the next smallest element from a structured bracket.

O(n log n) Heap Based Memory: O(n)
