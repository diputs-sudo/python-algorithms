Sorting algorithm

Tim Sort

A hybrid stable sort that combines insertion sort on small runs with merge sort over larger runs.

Tim Sort is designed for practical, real-world data. It first sorts small chunks called runs, then merges those runs together. Real industrial Tim Sort implementations add even more optimizations, but the core idea is still “small-run sorting plus stable merging.”

Step by Step

  • Split the array into small runs.
  • Sort each run using insertion sort.
  • Merge neighboring sorted runs.
  • Repeat merges until the whole array is one sorted run.

Usage Notes

Use it when

You want a practical stable sort idea that performs well on partially ordered data.

Good to know

Tim Sort gets much of its practical power from exploiting existing order in the data, not just from merging alone.

Explanation

How Tim Sort Works

Tim Sort is a hybrid of insertion sort and merge sort. It handles small runs with insertion sort because that is very efficient on tiny arrays, then uses stable merges to combine those runs into larger sorted blocks.

Real Tim Sort implementations are more advanced than this educational version, but the main lesson is still useful: practical algorithms often mix multiple strategies instead of relying on only one idea.
