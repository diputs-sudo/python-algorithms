Sorting algorithm

Merge Sort (In Place)

A memory-conscious merge variant that avoids an extra merge buffer by shifting values inside the original array.

In-place merge sort keeps the divide-and-conquer structure of Merge Sort, but replaces the normal temporary merge buffer with repeated shifting inside the array. That saves merge memory, but it makes the merge step much more expensive and harder to implement.

Step by Step

  • Split the array just like regular Merge Sort.
  • Sort the left and right halves recursively.
  • When a right-half value belongs earlier, save it and shift the left block right.
  • Insert that value into the open slot without using a separate merge array.

Usage Notes

Use it when

You want to explore advanced in-place merging ideas and memory-performance tradeoffs.

Avoid it when

You just need a practical Merge Sort. The standard buffered version is usually simpler and faster.

Explanation

How Merge Sort In-Place Works

Standard Merge Sort uses extra memory to merge two sorted halves. In-place merge sort avoids that extra buffer by rotating or shifting values inside the original array.

In this educational version, when a value from the right half should appear earlier, the algorithm shifts the left-side block one step to the right and inserts that value into the gap. That keeps the merge stable, but it can be expensive.

What Makes It Different

Regular Merge Sort trades memory for a clean merge step. In-place merge sort removes that extra buffer, but pays for it with much more complicated movement inside the array.

Pseudocode

if size <= 1:
    return

sort(left half)
sort(right half)

while left and right overlap:
    if leftValue <= rightValue:
        move left pointer
    else:
        shift left block right
        insert rightValue
