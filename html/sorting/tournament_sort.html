Sorting algorithm

Tournament Sort

A comparison sort inspired by knockout brackets, where a tournament tree keeps the current winner at the root.

Tournament Sort builds a binary tree of winners. Each internal node stores the smaller of its two children, so the root always tells you the current minimum. After extracting that winner, the tree is updated and the next winner rises to the top.

Quick Facts

Best: \(O(n \log n)\) Average: \(O(n \log n)\) Worst: \(O(n \log n)\) Memory: \(O(n)\) Tree Based Not Stable Selection Style
Best for

Learning how winner trees can repeatedly select the next smallest value efficiently.

Watch out for

It needs extra tree storage, so it is more educational than practical for everyday in-memory sorting.

Step by Step

  • Place values at the leaves of a tournament tree.
  • Each parent stores the smaller value from its two children.
  • The root becomes the current winner.
  • Remove that winner, update the path upward, and repeat.

Usage Notes

Use it when

You want to study winner trees and repeated minimum extraction in a structured way.

Avoid it when

You need a simple in-place sort. Tournament Sort uses extra memory for the tree.

Explanation

How Tournament Sort Works

Tournament Sort behaves like a bracket tournament where smaller values keep winning their matches. By the time the comparisons reach the root, the overall minimum has won the whole tournament.

After that minimum is extracted, the algorithm replaces its leaf with a sentinel and updates only the affected path back to the root. That is how the next winner is found efficiently.
