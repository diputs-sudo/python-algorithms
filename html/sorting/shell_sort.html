Sorting algorithm

Shell Sort

An optimized insertion-sort family algorithm that starts with large gaps and gradually reduces them to finish with a final gap of 1.

Shell Sort improves on plain Insertion Sort by moving values long distances early. Instead of comparing immediate neighbors only, it performs insertion-like passes over elements separated by a gap, then shrinks that gap until the array is nearly sorted.

Quick Facts

Best: Better than \(O(n^2)\) Average: Often around \(O(n^{1.5})\) Worst: \(O(n^2)\) Memory: \(O(1)\) In-Place Gap Based Not Stable
Best for

Medium-sized inputs where a simple in-place improvement over Insertion Sort is useful.

Watch out for

Its performance depends heavily on the chosen gap sequence, so it is less predictable than \(O(n \log n)\) sorts.

Usage Notes

Use it when

You want an in-place algorithm that moves values farther than Insertion Sort can early on.

Avoid it when

You need stable behavior or predictable \(O(n \log n)\) performance.

Explanation

How Shell Sort Works

Shell Sort is often described as “insertion sort with jumps.” Instead of moving elements only one position at a time, it first compares values that are far apart and gradually narrows the spacing.

Those early long-distance moves help get values much closer to where they belong. By the time the gap reaches 1, the array is often already partially organized, making the final insertion-style pass much cheaper.

What Makes It Different

Regular Insertion Sort only shifts neighboring values. Shell Sort adds a gap, which lets it move elements much farther in the early passes before tightening up into a normal insertion-style finish.

Pseudocode

gap = n // 2
while gap > 0:
    for i from gap to n - 1:
        temp = array[i]
        j = i
        while j >= gap and array[j - gap] > temp:
            array[j] = array[j - gap]
            j -= gap
        array[j] = temp
    gap //= 2
